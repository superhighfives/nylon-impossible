# Ordering

Currently checking and unchecking items causes them to jump back to where they came from. They should go to the bottom of the incomplete list.

## Problem

When a user checks off a todo, it currently stays in its current position or jumps back to its original position. This creates a confusing UX where completed items are mixed with incomplete items.

## Expected Behavior

When a todo is marked complete, it should move to the bottom of the incomplete list. If a todo is unchecked, it should return to its appropriate position based on the ordering algorithm (likely fractional indexing).

## Areas to Consider

- How this interacts with fractional indexing and manual reordering
- Whether completed items should have their own section (like the iOS app)
- Animation/transition behavior when items move
- Performance with large todo lists
