# Ordering

Checking and unchecking items caused them to jump back to where they came from. Fixed so unchecked items move to the end of the incomplete list.

## Problem

When a user unchecked a todo, it would snap back to its original position (based on its fractional index), creating a confusing UX where the item would jump away from where the user tapped.

## Solution

Modified `toggleTodo` in `TodoViewModel` to accept the full `allTodos` list. When unchecking a completed item, it now generates a new fractional index position after the last incomplete item, so the todo lands at the bottom of the active list instead of its original position.

## What Was Done

- Updated `toggleTodo(_ todo:, allTodos:)` signature in `TodoViewModel`
- When unchecking, filters incomplete todos, sorts by position, and assigns a new position after the last one via `generateKeyBetween`
- Updated call site in `ContentView` to pass `todos`
- Added `toggleTodoUncheckedMovesToEnd` test
- Fixed CI: simulator discovery for `macos-26` runner (iOS runtimes not pre-installed)
- Fixed CI: invalid fractional index key `"b0"` in test (correct next key after `"a0"` is `"a1"`)
- Fixed CI: added `@MainActor` to `toggleTodo` tests that mutate `@Model` properties off the main thread
