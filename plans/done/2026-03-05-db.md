# Database Management Scripts

**Date**: 2026-03-05
**Status**: Complete

## Overview

Added comprehensive database management scripts for local D1 development. The scripts provide utilities for resetting, seeding, backing up, and checking the status of the local SQLite database used by both the web and API packages.

## Architecture

### Key decisions

- **SQL-based seeding**: Used raw SQL seed file instead of TypeScript for simplicity and direct wrangler integration
- **Shared database**: Both web and api packages use the same database, so scripts are mirrored across both
- **Local-only operations**: Backup and reset only work on local SQLite files in `.wrangler/state/`
- **Wrangler integration**: All commands use wrangler CLI for consistency with existing migration workflow

## Files

| File | Purpose |
|------|---------|
| `src/web/src/db/seeds/seed.sql` | SQL seed file with test user, lists, and todos |
| `src/web/package.json` | Local database scripts for web package |
| `src/api/package.json` | Local database scripts for api package |
| `package.json` | Root-level convenience commands |

## Scripts

### Available Commands

```bash
# Fresh database (reset + migrate + seed)
pnpm db:fresh

# Reset database (drop + recreate with migrations)
pnpm db:reset

# Seed with test data
pnpm db:seed

# Backup local database
pnpm db:backup

# Check migration status
pnpm db:status
```

### Per-package Commands

All commands available at root and per-package:

```bash
pnpm web:db:fresh      # Web package
pnpm api:db:fresh      # API package
pnpm db:fresh          # Root (runs web version)
```

## Seed Data

The seed script creates:

- **User**: test@example.com (ID: user_test_123)
- **Lists**: TODO, Shopping, Bills, Work
- **Todos**: 5 sample tasks with various priorities and completion states

## Backup Storage

Backups stored in `.wrangler/backups/` with timestamp:
- Format: `db-YYYYMMDD-HHMMSS.sqlite`
- Automatically gitignored via `.wrangler/` in `.gitignore`

## Usage Example

```bash
# Get fresh database for testing
pnpm db:fresh

# Backup before major changes
pnpm db:backup
# Output: Backup created: .wrangler/backups/db-20260305-222518.sqlite

# Check what migrations are pending
pnpm db:status
```
