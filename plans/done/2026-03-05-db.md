# Database Management Scripts

## Context

We use Cloudflare D1 (SQLite) with Drizzle ORM across two packages (`web` and `api`). Currently we have basic migration commands but lack comprehensive database management utilities for local development.

**Current state:**
- Local D1 runs via Miniflare with SQLite files in `.wrangler/state/v3/d1/`
- Both packages share the same database
- Existing scripts: `db:migrate`, `db:migrate:remote`, `db:generate`, `db:studio`, `db:push`

## Proposed Scripts

### 1. `db:drop` / `db:reset`
Drop and recreate the local database.

```bash
# Drop local D1 SQLite database
rm -f .wrangler/state/v3/d1/miniflare-D1DatabaseObject/*.sqlite*

# Re-apply migrations to get clean slate
pnpm db:migrate
```

### 2. `db:seed`
Seed the database with test data for development.

**Needs:**
- Create `src/web/src/db/seed.ts` script
- Insert test users, lists, todos
- Use Drizzle ORM with local D1 binding

### 3. `db:backup`
Backup local database before destructive operations.

```bash
# Create timestamped backup
cp .wrangler/state/v3/d1/miniflare-D1DatabaseObject/*.sqlite \
   .wrangler/backups/db-$(date +%Y%m%d-%H%M%S).sqlite
```

### 4. `db:status`
Show database status and pending migrations.

```bash
# List applied vs pending migrations
wrangler d1 migrations list nylon-impossible-db --local --persist-to .wrangler/state
```

### 5. `db:validate`
Validate schema matches the database (dry-run check).

```bash
# Could use drizzle-kit push --dry-run if supported
# Or compare migration files against _drizzle_migrations table
```

### 6. `db:fresh`
Complete reset: drop + migrate + seed.

```bash
# Combo command for fresh database
pnpm db:drop && pnpm db:migrate && pnpm db:seed
```

## Implementation Priority

1. **High:** `db:reset` - Most immediately useful for development
2. **High:** `db:seed` - Essential for testing with realistic data
3. **Medium:** `db:backup` - Safety net before migrations
4. **Low:** `db:status`, `db:validate` - Nice to have diagnostics

## Implementation Summary

**Status:** ✅ Completed

### Scripts Added

All scripts available at root level (`pnpm <script>`) and per-package:

- **`db:reset`** - Deletes local SQLite database and re-applies migrations
- **`db:seed`** - Seeds database with test user, 4 default lists, and 5 sample todos  
- **`db:fresh`** - Combo: reset + seed in one command
- **`db:backup`** - Creates timestamped backup in `.wrangler/backups/`
- **`db:status`** - Shows pending migrations

### Files Created

- `src/web/src/db/seeds/seed.sql` - SQL seed file with test data

### Usage

```bash
# Fresh database with test data
pnpm db:fresh

# Check migration status
pnpm db:status

# Backup before destructive changes
pnpm db:backup

# Reset to clean state (migrations only)
pnpm db:reset

# Seed existing database
pnpm db:seed
```

### Test Data Included

- **User:** test@example.com (ID: user_test_123)
- **Lists:** TODO, Shopping, Bills, Work
- **Todos:** 5 sample tasks across different lists with various priorities/completion states

### Notes

- Both `web` and `api` packages have identical scripts since they share the same database
- Backups stored in `.wrangler/backups/` (gitignored by default via `.wrangler/` in gitignore)
- Seed file uses SQL rather than TypeScript for simplicity and direct wrangler integration
